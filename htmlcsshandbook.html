Basics Of HTML

Put the code that you want to show on your webpage in the <body> area.

Layout

<!DOCTYPE html>

<html>

                <head>

                        <title>Example Title</title>

                        <style>

        </style>

                </head>

        <body>

        

        </body>

</html>

Always end with </syntax that you used> so <p>Example</p>

<p> - Paragraph

<br> - Page Break

<b> / <i> - Bolds or Italics it

<th> - Table Header

<tr> - Table Row

<tbody> - Groups Table Body (Not Need)

Styling With Inline CSS[2]

Syntax

When you want to change the style you put style=”example:excolor;”.

So if you want to change the color of the <p> you do

<p style=”color=purple;”>Text Here</p>

Result

Text Here

There are many types of styles but the main ones you’ll use are…

  • color - Changes text color
  • background-color - Changes Background Color
  • font-size - Changes Text Size
  • font-family - Changes Font
  • More at https://www.w3schools.com/cssref/index.php

You can also add multiple styles to one syntax.

<p style=”color=blue;” “font-size=20px;”>Example Text</p>

Result

Example Text

This is the simplest way to style text in HTML, but it is not the most efficient way to style medium or big websites, for that you should use CSS which is a different type of coding language within HTML.

Styling With External CSS

Basics

Although you can use HTML styling it is really important and recommended that you know CSS styling. When you want to style with CSS you use the <style> syntax in the <head> of the code.

<!DOCTYPE html>

<html>

                <head>

                        <title>Example Title</title>

                        <style>

                                CSS CODE GOES HERE

        </style>

                </head>

        <body>

        

        </body>

</html>

As previously mentioned CSS is a different type of coding language so it has a different syntax.

Syntax

<!DOCTYPE html>

<html>

                <head>

                        <title>Example Title</title>

                        <style>

                                h1 {

                                color:blue;

}        

        </style>

                </head>

        <body>

        

        </body>

</html>

Now with this code each h1 (heading 1) will be blue. We no longer need to put style=”color:blue;”.

Tag, Class, ID

There are 3 ways to apply style with CSS, one is with tags which is the one that we just did, similying controlling one entire parameter. For example the last code would turn every single heading 1 to blue. But what if you only wanted one heading to be blue with CSS? That's where Class and ID come into play. They are mostly the same except that ID takes priority over Class which we’ll take over in the next section. Let's go over Class first.

Class is written as follows in the <style> area:

.anything {

        color:blue;

}

As you can see it is mostly the same but except for the syntax at the beginning you use anything after the . can be any text you want as long it's no caps and no spaces. You apply this with the following:

<p class=”anything”>This is blue<p>

So if you used this the text here would be blue due to the class at the beginning.

Now, ID is very similar to classes except that you use a # instead of a . and you use id=”anything[3]” to apply it. So if you wanted blue text with CSS’ ID you would use this code:

<!DOCTYPE html>

<html>

                <head>

                        <title>Example Title</title>

                        <style>

                                #blue {

                                Color:blue;

}        

        </style>

                </head>

        <body>

                <p id=”blue”>This text is blue</p>

        </body>

</html>

Cascade

Now what if something is contradictory in your styling? For example:

<!DOCTYPE html>

<html>

                <head>

                        <title>Example Title</title>

                        <style>

                                p {

                                color:red;

                                }

                                #blue {

                                color:blue;        

                                }

        </style>

                </head>

        <body>

                <p id=”blue”>This text is blue</p>

        </body>

</html>

What will the output be? Well it would be blue due to the cascade!

The cascade is a ranking system which helps if something is contradictory. Here is the ranking for the cascade:

  1. !importent Tag
  2. ID
  3. Class
  4. Tag
  5. Inline CSS
  6. Order

The !importent tag goes right after one of the styles. For example:

h1 {

        color:blue !important;

}

Now every single h1 will be blue even if it has an ID tag saying otherwise. Also again, please keep in mind that this is only for contradictory statements, if one says to be bold and another says to make it blue it will do both, this is only if something can be only one thing, like color. Next, 2-5 are easy to understand; they are just the type of Tag you are using. Order though is a bit weird, it is the opposite way the code is written. (See example on next page)

Example

<!DOCTYPE html>

<html>

                <head>

                        <title>Example Title</title>

                        <style>

                                .red {

                                color:red;

                                }

                                .blue {

                                color:blue;        

                                }

        </style>

                </head>

        <body>

                <p class=”blue red” >Is this text blue or red?</p>

        </body>

</html>

The text here would be blue since it is the last one in <style>. It is a bit weird but that’s HTML!

Multi-File Website

Syntax

Now since you know the basics let's get into a bit more detail and the complex stuff.

Since you can’t really show a multi-file website we will just show it here and won't use it after that.

There are 2 main types of files you can have in a website, .html and .css. HTML is the content and CSS is the code you would normally put in the <style> tag. The homepage of your website is always index.html, but the other you can name however you want, although it is recommended to name the CSS one style.css. Here is an example for style.css:

h1 {

        color:blue;

}

That would be all you would put in the CSS file. If you wanted now to add it to one of html files you would put:

<link rel="stylesheet" type="text/css" href="style.css">

In the <head> portion of the html file. This code connects the style.css to the code and tells the code to use it as the <style> tag.

So you have your index.html and style.css and want to add another .html, but what does another .html file even do? It is basically like another page for your website so you can separate the content. So you have your whatever.html with all the same syntax as any other basic .html file, now you want to connect it to your index.html. It's as simple as adding a link! You would add a link like to a normal website but instead of using https:/example.com you use whatever.html!

<a href = “whatever.html”>Whatever Page</a>

Now this link when the user clicks on it will take them to the other page![4] That’s it, now get to building large and good looking websites, and have some fin with HTML!
